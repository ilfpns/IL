- 정리
    - 레이어
        - 신경망의 기본 구성 단위이다.
        - 입력층 → 은닉층 → 출력층
    - 히든 레이어
        - 신경망 기본 구성 단위로
        - 은닉층이 많을수록 더 복잡한 패턴을 학습 할 수 있지만, 과적합 문제가 생길 수 있다
    - AI 결과 표 검증
        - 모델 검증을 위해서 혼동행렬, 정밀도/재현율 표, 예측값vs실제값 등을 시각화해서 놓치는 부분을 검증 할 수 있다
    - MLP
        - 기본적인 완전연결 신경망이다
        - 모든 뉴런이 다음 레이어의 모든 뉴런과 연결된다
    - 시계열 데이터 검증 가능성
        - 일반 랜덤 분할을 쓰면 미래 데이터가 과거 학습에 섞이는 데이터 누수 발생
        - 시간을 지키는 Walk-forward validation, TIme-Series split등으로 검증을 해야 한다
    - 정확도는 유지하되, 양자화랑 피처 엔지니어링을 잘 해야 함
        - 양자화(Quantization): float32 → int8 등으로 모델 가중치 압축. 정확도 손실을 최소화하면서 모델 크기↓, 추론 속도↑. 엣지 디바이스(센서 장비 등)에 필수
        - 피처 엔지니어링: 원시 데이터에서 모델이 학습하기 좋은 의미있는 변수를 만드는 작업 (예: 이동평균, 차분, 시간대 분리 등). 모델 성능에 가장 큰 영향을 미치는 단계
    - 피처별 중요도를 나눠서 피처링 엔지니어 가능
        - 각 입력 변수가 예측에 얼마나 기여하는지 수치화
        - 방법으로는 SHAP, Permutation Importance, 트리 계열 모델의 내장 feature_importances_ 등이 있다,  중요도 낮은 피처를 제거하면 모델 단순화 + 성능 유지 가능
    - Epoch
        - 전체 학습 데이터를 한 번 다 돌리는 단위이다
        - Epoch 10이라면 데이터 전체를 10번 반복 학습한다

### 피처 설정

다음 문제로를 위해서 다음과 같은 설정을 진행해야 한다. 

- 피처의 설정
    
    다음 두 과제를 하는 것이 피처 엔지니어링이다.
    
    - 피처 수 정하기
    - 피처를 줄이면서 정확도 유지
    
    모델의 좋은 성능은 대부분은 좋은 알고리즘보다 좋은 피처에서 나온다는 말이 있다.
    
    데이터를 모델이 해석할 수 있는 구조로 바꾸는 피처 엔지니어링 기술이 중요하다
    
    - 목표
        - **예측력을 높이기 위한 정보 구조화**
        → AI나 프로그램이 데이터를 더 잘 이해하도록 정리하는 것.
        예를 들어 학생 성적 데이터를 그냥 글로 두는 대신`출석률`, `과제 점수`, `시험 점수`처럼 항목별로 나누면
        어떤 요소가 성적에 영향을 주는지 더 잘 예측할 수 있음.
        
        ---
        
        - **복잡한 관계를 단순화**
        → 너무 복잡한 데이터를 이해하기 쉬운 형태로 바꾸는 것.
        예를 들어 사람 키·몸무게·나이·운동량 등 수십 개 정보를 그대로 쓰면 복잡하니까,
        이를 `BMI` 같은 하나의 값으로 묶어 단순하게 표현할 수 있음.
        
        ---
        
        - **비정형/범주형 데이터를 수치화**
            
            → 글자나 종류 데이터를 숫자로 바꾸는 것.
            
            컴퓨터는 숫자를 더 잘 처리하기 때문임.
            
            예시:
            
            - 남자 / 여자 → 0 / 1
            - 빨강 / 파랑 / 초록 → 1 / 2 / 3
            - “좋아요”, “별로예요” 같은 리뷰 → 감정 점수 숫자화
        
        ---
        
        - **모델 입력 조건을 맞추기 위한 스케일 정규화**
            
            → 데이터 값 크기를 비슷하게 맞추는 것.
            
            어떤 값은 너무 크고 어떤 값은 너무 작으면 AI가 헷갈릴 수 있음.
            
            예시:
            
            - 키: 170
            - 연봉: 50000000
            
            이렇게 차이가 너무 크면 연봉만 중요하게 인식될 수 있어서
            
            둘 다 0~1 사이 값처럼 비슷한 크기로 변환함.
            
            - 키 → 0.68
            - 연봉 → 0.7
    
    단순하게 생각하면 너무 많거나 불필요한 피처를 줄이면서 정확도를 유지하는 것이 중요하다
    

### AI 종류

이제 어떤 AI를 쓸지 정해야 한다. 

시계열 데이터를 위해서는 LSTM 등의 모델을 쓸 수 있고, 임베디드 환경에서는 TinyML 등을 사용 할 수 있다.
또한 Inference 모델을 쓸 것인지, Reasoning 모델을 쓸 것인지 정해야 한다.

| **구분**  | **인퍼런스 (Inference)** | **리즈닝 (Reasoning)** |
| --- | --- | --- |
| **개념** | 학습된 데이터를 기반으로 즉시 결과를 뱉어내는 과정 | 논리적 규칙에 따라 해답의 풀이 과정을 단계별로 전개하는 과정 |
| **비유** | 구구단을 보자마자 바로 답을 툭 뱉어내는 것 | 수학 서술형 문제의 풀이 과정을 차근차근 적어 내려가는 것 |
| **특징** | 속도가 빠르고 효율적임 | 속도는 느리지만 오류가 적고 논리적인 문제 해결에 탁월함 |
| **목적** | 실시간 응답, 데이터 분류 및 예측 | 복잡한 수학, 논리, 코딩, 추론 모델 작업 수행 |
- 인퍼런스: 학습한 그대로 추론 값을 그대로 내놓음
- 리즈닝: 추론을 하면서 모델 자체가 다시 생각을 하면서 결론 값을 도출함
