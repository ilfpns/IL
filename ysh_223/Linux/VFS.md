> **Virtual Filesystem**
[[Linux]]
---

⇒ 애플리케이션과 파일시스템 사이의 가상적인 층으로 애플리케이션이 다얀한 파일시스템에 일관된 방식으로 접근 하도록 도와준다

⇒ 앱이 접근하는 파일이 어느 파일시스템에 저장되었는지 고려하지 않고 일관된 표준 POSIX 인터페이스를 이용해 파일에 접근하는 것을 가능케 한다

한 마디로 앱은 어떤 함수를 실행할지 정하지 않고 일단 호출하면 VFS가 알아서 지정해줌

> **VFS 객체**

---

⇒ VFS는 다양한 파일 시스템과 데이터를 주고받기 위해 4개의 객체를 이용한다

- super block
    
    ⇒ 파일 시스템이 마운트될 때, VFS는 super_block을 하나 만들고. 해당 파일시스템은 자신의 동작 방식을 super_block의 s_op에 등록한다.
    
    ⇒ 이후 VFS는 s_op를 통해 파일시스템의 고유 동작을 수행한다
    
    ⇒ 마운트된 파일시스템의 정체성과 동작 방식을 담은 객체
    
- inode
    
    ⇒ 이름을 제외한 파일의 메타데이터 객체이다
    
- dentry
    
    ⇒ 이름과 inode를 연결하는 개체이다, 경로에서 이름을 inode에 매핑한다
    
- file
    
    ⇒ 열린 파일 상태로, 프로세스가 open한 사용 중인 파일 객체이다
    

file은 “누가 어떻게 쓰는지”를 나타낸다.

dentry는 “어떤 이름인지”,

inode는 “무엇인지”,

super_block은 “어디에 속했는지”,
