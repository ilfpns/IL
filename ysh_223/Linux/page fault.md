> **Page fault**
[[OS]]
---

⇒ 어떤 프로세스가 페이지를 쓰려 할 때 디스크에 있는 페이지를 RAM으로 가져옴

⇒ 이 때 가져오기를 실패해서 가상 페이지 주소에 대응되는 물리 페이지가 RAM에 존재하지 않을 떄 발생하는 이벤트

- Demand Paging
    
    필요할 때 디스크에서 RAM으로 페이지를 올리는 방식
    

### 특징

- 가상 페이지 기준으로 발생한다
- OS 개입이 발생한다
- 성능 비용이 크다
- 종류가 있다
    - Minor page fault : RAM안에 있지만, 매핑이 없다
    - Major page fault : 디스크에 접근 필요

### 목적

1. 프로세스가 가상 페이지 접근
2. 해당 페이지의 물리 프레임이 RAM에 없음
3. **page fault 발생**
4. **커널 page fault handler 실행 (트리거됨)**
5. 디스크/스왑 → RAM 페이지 적재
6. 페이지 테이블 갱신
7. 명령 재시작

⇒ 4번의 발생 조건은 page fault

⇒ 그런 page fault를 이용하여

⇒ 필요한 페이지만을 RAM에 올리는 트리거로 쓴다
