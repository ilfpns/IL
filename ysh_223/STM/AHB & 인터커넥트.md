# 버스
⇒ CPU에서 메모리나 입출력 기기에 데이터를 송출하거나 반대로 데이터를 받을 때 사용하는 전송롤 
### **AMBA (Advanced Microcontroller Bus Architecture)**
⇒ ARM사의 BUS protocol이다
⇒ 세대별 + 용도별 프로토콜의 집합이 총 4개이다
⇒ 트래픽 격리 + 전력 + 구조 단순화를 위해 사용된다

- AH(High)B
    ⇒ 고속으로 동작하는 장치들이 연결된 버스
    ⇒ multiplex bus로 주소라인, 제어라인, 데이터 라인 모두 공유한다
    ⇒ 주소 → 데이터 순서로 전송
  
- AS(System)B
    ⇒ 고속으로 동작하며 AHB와 달리 Rising Edge와 Failing Edge 모두를 사용한다
    ⇒ 주소, 제어, 데이터 라인이 모두 분리된다
    ⇒ 현재 잘 쓰이지 않는다

- AP(erippehral)B 
    ⇒ 비교적 느린 속도의 주변장치를 제어한다. 전력 소모를 위해 간단한 인터페이스를 가진다
    ⇒ 주소 & 파이프 라인 분리된다

- AXI 
    ⇒ 다중채널 버스로 읽기/쓰기에 최적화됨.
    - 채널 도입이 독립적이다
    - 채널의 제약이 없어 더 빠르게 전보를 전달할 수 있다
    - 병렬 처리가 가능하다, 보다 높은 처리량 제공


⇒ 버스가 3개인 이유는 각 장치마다 속도가 다르기 때문.

```cpp
-------A-------B-------C-------D------- . . .
// AHB에서 A가 저속 장치라면
// B까지 기다려야함 => 고속 장치의 이점을 살릴 수 없음
```

### **인터커넥트**
⇒ SoC 내부에서 여러 마스터와 슬레이브를 연결하는 하드웨어 통신 인프라
⇒ 데이터가 지나가는 길이다

**사용 이유**
- 여러 접근 동시 처리
- 느린 장치가 고속 장치를 방해하지 않음
- 우선순위 & 대역폭을 제어할 수 있음
⇒ 성능, 동시성, 실시간성을 위해 사용

**역할**
⇒ 연결 (Connectivity)
- CPU ↔ DRAM
- DMA ↔ Peripheral
- GPU ↔ Memory
- 모두 이어줌
