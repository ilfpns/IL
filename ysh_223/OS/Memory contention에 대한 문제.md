### Memory Barrier

컴파일러가 명령어 진행하는 강제 명령어 재배치를 막는다

- 컴파일러는 최적화를 수행하며, 코드의 실행 순서가 바뀌어 의도치 않은 결과가 도출 될 수 있다
- 이런 컴파일을 고정하고 코드 실행 순서를 고정하는 것이 메모리 배리어이다

다음과 같은 경우에 Memory Barrier가 구현되어 있다 ( = 하드웨어 개념이다)

- Volatile
- atomic
- lock

### Memory Concurrency/Consistency

임베디드 메모리 관점에서 왜 동시성/일관성이 중요할까?

⇒ 같은 메모리에 대해 다릅 접근/행동을 하면 이로 인해 발생한 오류를 잡아내기 어렵다.

#### Memory Concurrency 메모리 동시성

“동시적 수행이다” 이게 무슨 의미일까?

동시성은 여러 작업을 겹치게 수행하는 것이다. 실제로는 동시에 일어나지 않고 빠르게 작업을 전환하며 마치 동시에 처리하는 것처럼 보이게 만드는 기법이다.

이에 반대되는 개념으로 “병렬성”이 있다

#### 메모리 동시성

⇒ 동일한 메모리 공간에 동시에 접근할 때 발생하는 현상과 이를 제어하는 원리

- 경쟁 상태에 대해 이를 제어하는 원리임 ( EX : Memory Barrier)

그렇다면 이런 메모리 동시성을 해결하는 방법이 뭘까?

1. Atomic한 코드를 보장하면 된다
2. 다른 스레드에서 바뀐 값을 바로 볼 수 있는 visibility가 필요하다
3. Memory Consistency/Barrirer를 사용하면 된다

#### Memory Concurrency는 뭘까? ( = 메모리 일관성)

먼저 Memory Coherence라는 개념이 떠오를 것이다 (떠오르지 않는다면 더 공부하자)

이는 메모리 일관성이라고 한다. 그렇다면 같은 이름으로 불리우는 이 두 개념은 무슨 차이점을 가질까?

- Consistency ( = 일관성)
    
    ⇒ 타당한 방식이 아니더라도 항상 동일한 방식으로 수행되어야 한다
    
    ⇒ 어떤 동작을 수행함에 있어서 항상 동일한 방식으로 수행된다
    
    - 메모리에 작업하는 흐름에 대한 모델
    - A → B에 대해 다른 접근자들도 같은 흐름으로 접근함
- Coherence ( = 일관성)
    
    ⇒ 동일한 방식을 넘어 타당한 방식으로 수행되어야 한다
    
    - DMA가 바꾼 data를 CPU는 다른 값을 볼 수도 있음, 이 때 일관성이 지켜지지 않았음
    

타당한 방식이라는 말은 무슨 의미일까

사용하신 정의에서 Consistency가동일한 방식이라 한다면 시스템이 정한 규칙(예: Relaxed 모델)에 따라 무조건 일관되게 행동한다는 뜻이고, Coherence가 타당한 방식이라 한다면, 사용자가 어디서 데이터를 읽든 최신의 가장 타당한(올바른) 값이 나와야 한다는 의미이다
