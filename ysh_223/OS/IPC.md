### Inter Process Communication
[[OS]]
⇒ 프로세스들이 데이터를 서로 교환할 수 있는 통신 기법

- 독립적 프로세스
    
    ⇒ 실행 중인 다른 프로세스들과 데이터를 공유하지 않는 프로세스
    
- 협력적 프로세스
    
    ⇒ 실행 중인 다른 프로세스들에 영향을 주거나 받는 프로세스
    

**협력을 허용하는 환경을 제공하는 이유**

- 정보 공유 : 동일한 정보에 병행적으로 접근할 수 있는 기회
- 계산 가속 : 빠른 계산을 위해 서브 태스크로 나누어 병렬로 실행하게 하는것 (멀티코어 기준)
- 모듈성 : 기능을 별도의 스레도들로 나누어, 모듈식 형태로 구성

**IPC 방법**

1. 공유 메모리 (Shared Memory)
    
    - 프로세스간 메모리 영역을 공유해서 사용할 수 있도록 지원하는 설비
    - 중계자 없이 곧바로 메모리에 접근
    - 가장 빠른 IPC
    
	    ![[Pasted image 20260316101541.png]]
1. 파이프 (Pipe)
    
    - 메모리 공간 (버퍼)를 생성하여 프로세스 통신
        
    - 파이프는 두개의 프로세스를 연결
        
    - 송/수신을 위해 두 개의 파이프로 이뤄짐 (반이중 통신)
        
        ![[Pasted image 20260316101520.png]]
    - 익명 파이프
        
        - 일반적인 파이프
        - 통신할 프로세스가 명확하게 알 수 있는 경우 사용
        
        ```c
        > 부모-자식 or 형제 프로세스 간 통신에 사용
        > 외부 프로세스에서 사용 불가
        ```
        
    - 네임드 파이프
        
        - 전혀 모르는 상태의 프로세스들 간의 통신에 사용
        - 익명 파이프의 확장된 상태
        - 프로세스 통신을 위해 이름이 있는 파일을 사용
        - FIFO라 불리는 특수 파일을 이용해 서로 관련 없는 프로세스 간 통신에 사용
        - mkfifo 함수로 생성
2. 소켓 (Socket)
    
    - 컴퓨터 네트워크를 경유하는 프로세스 간 통신
    - 클라이언트와 서버가 소켓을 통해 통신하는 구조, 원격에서 프로세스 간 데이터를 공유할 때 사용
    - 전이중 (양방향) 통신
