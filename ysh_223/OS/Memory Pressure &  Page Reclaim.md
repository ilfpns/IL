## Memory Pressure [[OS]]

⇒ 시스템에서 사용 가능한 메모리가 부족해서 메모리 관리가 힘들어진 상태

## What is means

⇒ 메모리 자원이 거의 소진된 상태

즉, 물리 메모리가 부족해 OS나 런타임 메모리 할당을 처리하기 어려운 상황

### Characteristic

- 새 메모리 할당이 어렵다
    
- 스왑/페이지아웃 등 메모리 회수 작업이 자주 발생한다
    
- 메모리 관리 오버해드가 커진다
    
    - 오버헤드
        
        특정 기능을 수행하거나 작업을 처리하기 위해 직접적인 핵심 로직 이외에 추가적으로 소요되는 시간, 메모리 등 간접적인 비용
        
- 성능이 저하된다
    

### In OS…

운영체제에서는 메모리 관리가 시스템 성능과 안전에 직접적으로 관여된다

### How to solve

⇒ Page Reclaim

### What is Page Reclaim

⇒ 효율적으로 메모리를 회수하는 기술

페이지 회수 개념은 2가지가 존재한다

- ACTIVE 리스트 : 최근 자주 사용된 페이지
- INACTIVE 리스트 : 덜 사용되거나 오래된 페이지

**LRU 기반 접근 패턴**

⇒ OS는 PTE.accessed와 페이지 상태를 기반으로 페이지를 ACTIVE 와 INACTIVE 리스트 간에 이동시키며, 필요 시 비활성 리스트에서 희생 페이지 (Victim Page)를 선택해 회수

⇒ 즉, 오랫동안 쓰지 않은 페이지 먼저 종료함으로 효율적인 기법이다

- PTE.accesse
