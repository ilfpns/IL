#### 배경
[[CS]]
캐시는 hit와 miss 시에 데이터 접근 시간이 다르기 때문에 동작 시간을 정확하게 예측해야하는 몇몇 임베디드 시스템에서는 캐시의 사용이 문제가 된다

그렇기 때문에 TCM이 개발되었다

#### TCM

⇒ 캐시 사용으로 인한 처리시간 불확실성 제거

⇒ Tightly-Coupled Memory의 약자로 CPU에 바로 연결되는 고속 메모리 인터페이스를 의미한다

#### 특징

- 캐시와 다르게 별도 주소공간을 가지고 매핑되어있다
    
    ⇒ 빠른 접근이 필요한 데이터를 별도주소공간의 sram에 사용한다는 의미와 같다
    
- 캐시 영역에 포함되지 않는다
    

![[Pasted image 20260326194427.png]]
- ITCM : 코드 전용 고속 라인으로, CPU 실행 코드를 지연 없이 실행한다
- DTCM : 데이터 전용 고속 라인으로, 연산이 필요한 데이터를 지연 없이 전송한다

---

- AHBS : DMAC에서 CPU로 통하는 고속 버스
- AHBP : CPU가 주변 장치를 제어할때 사용하는 고속 버스
- AXIM : CPU가 대용량 외부 메모리에 접근할 때 Master 용도록 사용하는 고속 버스
- AHB Matrix : 버스 교차로
- AHB Bridge : 서로 다른 속도나 규격을 가진 도로를 이어주는 다리

---

- DMAC : CPU 대신 데이터를 옮겨주는 컨트롤러
- Low Latency Peripherals : 최소 지연을 달성해야하는 주변 기기
