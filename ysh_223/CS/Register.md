  Register는 무엇일까?
Register를 이제부터 R이라고 지칭하도록 하겠다.

R CPU내부에 위치한 작은 크기의 고속 메모리이다. 컴퓨터 내부에서 성능을 향상하는 데에 다양하게 사용되며, 이는 하드웨어 메모리 중 가장 빠른 속도를 보여준다.



임베디드 개발자가 R의 종류나 특징에 대해 모두 다 외우고 다닐 수는 없다. 이건 엄청나게 불필요한 일이다. 

하지만 적어도 5개의 R은 알아둬야 한다.



오늘은 이 5개의 R에 대해 정리해 보고자 한다.



먼저 레지스터는 번호를 매겨 절반으로 나누어 분리되어 있다.

R0 ~ R7까지는 어떤 모드에서 동작을 하더라도 똑같은 주소에서 데이터를 찾는다. 하지만 R8 ~R12는 FIQ동작을 할 때는 물리적으로 분리된 다른 주소에서 동작할 수 있다.



R0 ~ R7는 Unbanked registers라고 불린다.

R8 ~R14는 Banked registers라고 불린다.

1. Program Counter, PC (R15)
다음에 실행할 Instruction의 주소를 저장하는 R이다.


다음 사진은 어셈블리 코드와 해당 명령어에 매핑된 메모리 주소가 표로 정리되어 있다.

현재 IR (Instruction Register로 현재 실행 중인 메모리 주소를 저장함)에 위치한 주소가 0x100일 때, PC는 0x104를 저장하고 있을 것이다.

2. Stack Pointer, SP (R13)
함수가 호출될 때 돌아갈 함수의 주소를 가지고 있는 함수이다.

이를 어디에 적용할 수 있을까?


다음 사진을 보자.

재귀 함수 코드에서 함수는 계속해서 함수를 호출하고, 이 주소들을 어디에 저장하지 않으면 다시 돌아갈 수 없을 것이다.



2번 함수가 호출될 때 1번 함수의 주소를 담아간다. 그렇다면 2번이 종료되고 return 될 때, 1번의 함수 주소로 돌아갈 것이다.

즉, 스택의 최상단을 가리키는 것이 SP의 역할이다. 이렇게 우리는 SP만 안다면 함수가 어디까지 호출되었는지를 알 수 있다.



SP는 돌아갈 함수의 주소만을 가리키는 역할만을 하는 것이 아니다. 그런 역할만을 하는 R은 LR이라고 한다. SP는 기본적으로 Stack에 쌓이는 데이터에 대한 주소 포인팅을 모두 포함한다.

3. Link Register, LR (R14)
SP와 비슷하지만 함수가 종료되고 돌아가 함수의 주소만을 저장한다. 

함수 흐름의 핵심으로 돌아갈 주소를 잃어버린다면 Loop에 갇히게 될 것이다.

4. Combined Program Status Registe, xPSR
현재 프로그램 상태를 보여주는 레지스터로 다음 세 가지의 논리적 레지스터를 포함하여 작동한다.



APSR: Application Program Status Register
주로 산술 및 논리 연산의 결과(Flag)를 저장한다.
IPSR: Interrupt Program Status Register
현재 실행 중인 예외나 인터럽트의 번호를 나타낸다.
인터럽트 발생 시 해당 인터럽트 번호가 이 필드에 기록된다.
EPSR: Execution Program Status Register
프로세서의 실행 상태 정보를 담고 있다.
xPSR은 일반적인 ASM의 MOV로 읽거나 움직일 수 없고, 반드시 특수 레지스터 접근 명령어인 MSR과 MRS로 접근해야 한다.



- 읽기 : MRS R0, XPSR 

- 쓰기 : MSR XPSR, R0

5. CONTROL
시스템의 Privilege level과 SP를 결정하는 제어 레지스터이다. xPSR은 상태만을 보여준다면 CONTROL은 동작 방식을 설정한다. 



- Privilege level : 어떠한 시점에서 CPU가 파일이나 자원에 접근 할 수 있는 권한 범위를 나타낸다.



2개의 핵심 비트를 중심으로 동작한다.



1. nPRIV (Bit 0) : 권한 제어

    0 : 모든 리소스, 레지스터에 접근 가능

    1 : 일부 접근이 제한됨



2. SPSEL (Bit 1) : SP 선택

Cortex-M은 두 개의 SP를 가진다. 이때 어떤 것을 쓸지 이 비트가 결정한다.

    0 : 시스템 예외나 메인 루틴에서 사용

    1 : RTOS환경에서 Task의 스택으로 사용
