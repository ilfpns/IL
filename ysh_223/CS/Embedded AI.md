## **임베디드 AI**
[[CS]]
임베디드에서 AI를 사용하는 방법은 크게 3가지로 나뉜다

- 온디바이스
    
    <aside>
    
    - 외부와 네트워크 연결이 없다
    - 기기 자체의 프로세서에서 작동한다 </aside>
- 엣지 컴퓨팅
    
    <aside>
    
    - 온디바이스를 포함하여, 여러 대의 기기 데이터를 모아서 처리하는 “엣지 게이트웨이”, “로컬 서버” 까지 포함된다
    - 수십 개의 노드로부터 얻어온 데이터를 하나의 고성능 보드나 로컬 서버에서 처리하는 개념이다 </aside>
- 서버
    
    <aside>
    
    - 보드와 서버가 직접 통신함
    - 보드는 서버에게 값을 넘기고 서버에서 직접 AI연산을 진행한다 </aside>

다음과 같이 나뉘는 경계는

- 처리 범위
- 실행 위치

를 기준으로 나뉘어진다

## 학습 데이터 셋

3개의 방식 중 어떤 방식을 채택하더라도 결국 AI에게 학습시킬 “데이터”가 있어야 한다

학습시킬 데이터를 몹는 과정 역시 4가지 각기 다른 방법이 존재한다

- 공개 오픈소스 데이터셋
    
    <aside>
    
    - 기업이나 연구자들이 이미 얻어놓은 데이터를 사용
    - Kaggle, UCI, PhysioNet 등에서 데이터를 가져올 수 있다 </aside>
- 데이터 증강
    
    <aside>
    
    - 직접 수집한 적은 데이터를 인위적으로 추가하는 방법이다
    - 노이즈 추가, 크기/시간 조절 등의 방법으로 데이터를 인위적으로 추가하는 방법이다 </aside>
- 시뮬레이션 및 합성 데이터
    
    <aside>
    
    - 수학적 모델링으로 가짜 데이터를 만드는 방법이다
    - 직접 실험하기 위헙하거나 불이익이 발생할 수 있는 상황에서 사용된다 </aside>
- 웹 크롤링
    
    <aside>
    
    - 거시적인 데이터를 다룰 때 주로 사용된다
    - 웹상의 데이터를 가져와서 라벨링하는 방식이다 </aside>

다음과 같이 4가지 방법의 공통점은 다음과 같다

- 많은 양의 데이터를 필요로 한다
- 수학적 계산이 필요하다

## 데이터 정제 및 필터링

모아온 데이터는 보통 노이즈나 결측치가 있는 값이 섞여 있다

그렇기 때문에 이를 걸러내는 작업이 필요하다

- 결측지/이상치 처리 : 센서 통신 에러로 값이 비어있거, 갑자기 비정상적으로 튀는 값을 수정하거나 삭제한다
- 노이즈 제거 : 가속도, 자이로, 마이크 센서 등에서 발생하는 잔떨림, 백색 소음 등을 LPF등을 적용하여 데이터의 파형을 부드럽게 만든다

## 라벨링

AI에게 어떤 데이터 상황에서 해당 상태가 정답인지, 그렇지 않은 값인지 구분할 수 있도록 정답지를 달아주어야 한다

데이터셋 라벨링을 진행하는 방법 역시 여러가지가 존재한다

- 시계열 데이터
    
    ⇒ 특정 시간 구간 단위로 라벨을 매긴다
    

## 윈도윙 및 피쳐 추출

길게 이어지는 데이터를 메모리가 부족한 MCU를 위해 일정한 크기로 쪼개는 작업이 필요하다

- 피쳐 추출
    
    ⇒ 쪼갠 윈도우 안의 Raw Data 를 그대로 쓸 수 없기 때문에, 평균, 분산, 주파수 에너지 등 특징값 (Feature)만 뽑아내어 데이터의 크기를 줄이는 작업이다
    

## 스케일링 및 정규화

센서마다 값의 단위가 다르다

- 온도 : 20 ~ 36
- 가속도 : -2.0 ~ 2.0

다음과 같이 각기 다른 데이터를 모델에 넣으면, 학습에 실패할 수도 있다

그렇기 때문에 모든 피쳐 값을 0~1 사이 또는 평균 0, 표준편자 1로 압축해야 한다

나중에 모델을 보드에 올릴 때(C/C++ 구현 시), 여기서 사용한 스케일링 수식과 동일한 수식을 보드 코드에도 똑같이 하드코딩해 넣어야 한다

이 부분에서도 해당되지 않는 예외도 존재한다

- Tree 모델
    
    ⇒ 단순히 값의 대소 관계만 비교한다
    
    이런 특징은 값에 관계 없이 이를 더 작은 수로 압축을 해도 “크다/작다”의 기준은 변하지 않기 때문에 스케일링을 진행하지 않는다
    

## 데이터셋 분할

잘 다듬어진 데이터를 용도에 맞게 3등분을 한다

- 7:2:1
- 8:1:1

역할을 다음과 같다

- Train set (학습용, 7) : 모델이 실제로 보고 학습하는 데이터셋
- Validation set (검증용, 2) : 학습 중간중간 모델이 잘 학습중인지 테스트하는 데이터셋
- Test set (평가용, 1) : 학습이 완전히 끝난 후, 처음 보는 데이터에 대한 성능을 평가하는 데이터 셋

## 양자화

임베디드 보드에 완성된 AI를 넣기 전에 경량화 작업이 필요하다

대부분의 MCU는 FPU가 아예 없거나 빈약하다

<aside>

FPU란?

---

Floating point Unit의 약자로 소수점 연산 장치이다

</aside>

그렇기 때문에 다음과 같은 특징을 보인다

- 속도 차이 : 소수점 연산을 흉내 내는 것 보다, 정수연산을 처리하는 게 수십 배 빠르다
    
- 메모리 차이 : 모델의 크기를 줄여, 작은 보드에 들어갈 수 있도록 한다
    
- 의문점
    
    그렇다면 4byte float 보다 8byte int가 빠를까?
    
    ⇒ 그렇다 결론적으로 int가 float보다 빠른 것은 사실이다
    
    하지만 8byte (uint64_t)는 MCU에게 버거운 크기이다
    
    그렇기 때문에 다음과 같이 경량화를 목표로 함
    
    - float32 (4byte) 와 int64 (8byte) 보다 int8 (1byte) 사용
