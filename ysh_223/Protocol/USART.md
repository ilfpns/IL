### UART
[[Protocol]]
---

⇒ Universal Asynchronous Receiver/Transmitter

⇒ 병렬 데이터의 형태를 직렬 방식으로 전환하여 데이터를 전송하는 방식

- 직렬 통신 : 1→1→1→1 순서로 전송
- 병렬 통신 : 1, 1, 1, 1 을 동시에 전송

**UART 하드웨어 구조**

- 하나의 선으로 통신한다
    
    - TX : 송신
    - RX : 수신
    
    다음 두 선으로 통신한다
    
    ( + 동기식은 클럭을 맞추기 위해 3개의 선으로 통신한다)
    
- TX, RX 총 2개의 선을 사용하므로 각자 따로 행동할 수 있다
    
- UART는 Full-Duplex로 전이중의 특징을 가진다
    
    ⇒ 두 선이 각자 원할 떄 송수신이 가능하다
    

### Asynchronous 비동기는 뭘까?

먼저 Synchronous는 데이터의 동기화를 위해서 별도의 클럭 신호가 전송된다

해당 클럭에 맞춰 데이터 송수신을 진행하게 된다

비동기는 별도의 클럭 신호를 전송하지 않는다

오직 다음 두 비트로 이뤄진다

- start bit
- stop bit

이런 통신은 초당 n개의 데이터를 보내기로 정한 BaudRate에 따라 통신하게 된다

이런 동기는 start/stop bit가 필요하지 않아서 데이터 전송 효율은 좋지만, 별도의 클럭 핀이 필요하다

### 클럭에 맞춰 전송하는 “동기”는 무슨 의미가 있을까?

- stop bit를 가지지 않기 때문에 불필요한 오버헤드 없이 빠르게 통신할 수 있다
    
    - 한 패킷 (1byte)당 stop bit를 이용하여 구분한다
- 전자 부품의 미세한 BaudRate 설정 오차를 없애고 완벽하게 통신할 수 있다
    
- 실무에서는?
    
    동기 UART는 클럭핀을 사용하므로 총 3개의 핀을 사용함
    
    ⇒ 핀을 2개만 사용하는 UART를 더 많이 사용함
    

### UART는 어떤 비트로 이뤄질까?

비동기 UART를 기준으로 한다

- 시작 비트 : 통신의 시작을 의미하며 한 비트 시간 길이 만큼 유지한다
- 데이터 비트 : 5 ~ 8비트의 데이터 전송을 한다, 몇 비트를 사용할 것인지는 해당 레지스터 설정에 따라 결정된다
- 패리티 비트 : 오류 검증을 하기 위한 비트로 사용된다
- 종료 비트 : 통신의 종료를 알리며, 세가지의 정해진 비트 만큼 유지해야 한다
