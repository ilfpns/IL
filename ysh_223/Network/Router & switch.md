> **Router**
[[Network]]
---

⇒ LAN을 연결해주는 장치

⇒ 데이터를 전송할 때 최종 목적지까지 거쳐야 하는 많은 경로 중에 통신 경로를 정하기 위해 사용됨

⇒ 한 통신망에서 다른 통신망으로 데이터 패킷을 전송

⇒ IP를 읽고 최적 라우트를 선택

### Static Route

- 관리자가 네트워크에 대한 경로 정보를 직접 지정
- 관리자에 의한 라우팅 정보만을 참조

### Dynamic Route

- 대규모 네트워크에 사용
- 변경된 네트워크에 대한 정보를 자동으로 교환
- Routin table을 자동으로 작성해 관리자의 초기 설정만 필요로 한다
- 메모리를 더 많이 차지한다

### IGP

⇒ 동일 그룹 네트워크에서 라우팅 정보를 교환할 때 사용되는 라우팅 프로토콜

- IGP의 세분화
    - **D**istance Vector (거리 벡터): 목적지까지의 거리(Hop count)와 방향만을 고려한다.
        - 설정이 간단하지만 네트워크 규모가 커지면 속도가 느려진다. (예: RIP)
    - Link State (링크 상태): 네트워크 지도를 그려서 최단 경로를 계산한다.
        - 변화에 빠르게 대응하지만 계산량이 많아 CPU를 더 씁니다. (예: OSPF)
    - EGP (Exterior Gateway Protocol): 서로 다른 그룹(AS) 간의 통신에 사용된다. (예: BGP - 인터넷의 경로를 결정하는 핵심 프로토콜)

### 스위치

⇒ 준비된 경로로 프레임(데이터 패킷)을 넘겨주는 것

### Flow

1. 들어오는 프레임 소스 MAC주소를 보고
    
    ⇒ 포트 1번 A라는 PC를 생각함
    
2. 목적지를 모를 때는 모든 포트에 다 뿌림
    
3. 목적지를 알면 해당 포트로만 보냄
