## IP & MAC

### Internet Protocol

⇒ OSI 7계층 중 3계층에 위치하는 프로토콜이다

⇒ 네트워크상에서 서로를 식별하고 데이터를 정확하게 전달하기 위해 장치마다 부여된 고유 디지털 주소

⇒ 즉, 인터넷에 연결된 각 기기를 구별하기 위해 사용되는 고유 번호

**IP는**

⇒ 수신측 : 데이터를 패킷으로 나눠 전송함

⇒ 송신측 : 분해된 패킷들을 다시 조립하여 원래의 데이터로 변환함

### Media Access Control

⇒ 물리적인 네트워크 세그먼트의 통신을 위해 네트워크 인터페이스에 할당된 고유 식별자

⇒ 전 세게에 중복되지 않는 하드웨어 고유 식별 번호

### 작동 방식

- MAC : 네트워크 인터페이스에 할당된 고유 식별자
    
    ⇒ DataLink계층에서 미디어 엑세스 제어 프로토콜에 의해 사용됨
    
- IP : 네트워크 장치에 할당된 논리적 숫자 주소
    
    ⇒ 네트워크에서 장치를 식별하고 통신할 수 있도록 함
    

## Different

1. **OSI 계층 관점**
    
    |구분|MAC|IP|
    |---|---|---|
    |계층|L2 (Data Link)|L3 (Network)|
    |처리 장비|Switch|Router|
    |단위|Frame|Packet|
    
2. **주소 특성**
    
    |항목|MAC 주소|IP 주소|
    |---|---|---|
    |길이|48bit (6 byte)|IPv4: 32bit / IPv6: 128bit|
    |표현|`AA:BB:CC:DD:EE:FF`|`192.168.0.1`|
    |할당|제조사|DHCP / 수동설정|
    |변경 가능|거의 고정|네트워크 바뀌면 변경|
    |유효 범위|LAN 내부|LAN + Internet|
    
3. **사용 기준**
    
    ⇒ 내부망과 외부망에서 사용될 때 차이를 보인다
    
    - 내부
        - GateWay 필요 없음
        - Flow
            1. 목적지 IP확인
            2. 같은 서브넷일 때
            3. ARP로 IP → MAC 변환
            4. MAC 기반 전송
    - 외부
        - GateWay 필요함 (IP 필요)
        - Flow
            1. 목적지 IP가 다른 네트워크임
            2. 기본 GateWay로 IP를 보냄
            3. GateWay의 MAC을 ARP로 알아냄
            4. Frame은 GateWay MAC으로 전송
            5. Router는 IP를 기준으로 가음 홉 결정

**결론**

- IP : 어디로 갈지
- MAC : 누구에게 줄지
