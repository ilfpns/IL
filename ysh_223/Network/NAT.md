## Nerwork address Translation
[[Network]]
1. IP 패킷의 TCP/UDP 포트 숫자와 소스 및 목적지의 IP 주소 등을 재기록하면서 라우터를 통해 네트워크 트래픽을 주고 받는 기술
    
    ⇒ 같은 공유 IP를 공유하기 때문에 일련 포트 번호를 부여하고, 이를 기록함
    
2. 외부 공인 IP 하나를 여러 대의 기기가 나눠 쓸 수 있게 하는 기술
    
    ⇒ 사설 IP 3개가 하나의 공인 IP로 활동할 수 있음 (나눠쓴다)
    
3. 사설 IP를 공인 IP로 바꿔줌
    

### Private Network (사설망)

⇒ IPv4 중 특정 대역을 공인 인터넷이 아닌 가정, 기업 등의 한정된 공간에 사용한 네트워크를 의미한다

- 사설IP는 사설망(내부망)에서만 사용할 수 있다
- 공인망(외부망, 인터넷)에서 접근할 수 없다

![[Pasted image 20260313171154.png]]
SubnetMask가 지정된 공유기에게 사설IP를 할당받음

- 사설망에서 공인 인터넷으로 나가고자 할 때 자신의 출발지 IP를 사설 IP 그대로 쓸 수 없음
    - 다른 사설망에 나와 같은 IP가 존재하므로, 구분할 수 없음

⇒ 사설IP를 공인IP 로 변환하여 사용하자! (NAT)

---

### 과정

![[Pasted image 20260313171207.png]]
1. 사용자는 웹에 접속하기 위해 NAT 장비 (Gateway)에 패킷을 보냄, 이는 1번과 같다
2. 이를 받은 NAT은 자신에게 허용된 규칙을 확인하고, 공인망에 보내기 위해 사설 IP를 자신의 공인 IP로 변환하여 웹서버에 전달, 이는 2번과 같다
3. 웹서버가 사용자가 보낸 요청을 처리하고 응답을 보냄, 패킷의 출발지와 목적이가 바뀜, 이는 3번과 같다
4. NAT 장비가 과거 사용자가 보낸 요청에 대한 응답입을 기억한다, 목적지 IP를 공인 IP에서 사설 IP로 변환하여 전달한다, 이는 4번과 같다

Exception

- 만약 같은 port를 사용한다면?
    - PAT를 사용
    - 출발지 포트를 겹치지 않도록 임의로 변경

### 위치

- 주로 관문 역할을 하는 Gateway 네트워크 장비가 NAT을 수행함
- 공유기 → 방화벽, VPN, L4 스위치 등 보안이 중요한 위치에서 수행됨

⇒ NAT 실행 장비는 설정된 규칙에 따라 허용/거부 판단을 실시함

⇒ 이를 통해 NAT을 실행하고, 기록해둠

- 수행 장비 : Session 장비 (이하 세션 장비)
- 기록 테이블 : Session Table (이하 세션 테이블)

### 관점

- IP와 IP (1:1) : Static NAT
- IP들과 IP들 (n:n) : Dynamic NAT
- 포트 포함 : NAPT
- Source IP (출발지) 변환 : SNAT
- Destination IP (목적지) 변환 : DNAT
