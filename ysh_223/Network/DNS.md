# **Domain Name System**

> 사용자가 입력한 도메인에 알맞는 IP를 찾아주는 것

찾아준다?

⇒ 사용자 입력 :

[google.com](http://google.com/)

DNS : [google.com](http://google.com) 도메인을 IP 형태로 변환 142.250.206.14  
→ 142.250.206.14로 접속

IP형태는 웹 페이지의 주소이지만 사용자가 실사용하기 어려움

→ 사용자는 외우기 쉬움 도메인을 사용

→ 이런 도메인을 DNS 방식으로 IP로 바꿔줌

### 구조

DNS는 총 3개의 구조로 이뤄짐

1. Domain name space
    
    → DNS가 저장하고 관리하는 계층적 구조
    
    →도메인 네임 스페이스 라는 규칙으로 도메인 이름 저장을 분석함
    
2. Name Server
    
    → 권한 있는 DNS 서버
    
    → 해당 도메인 이름의 IP주소를 찾는다
    
3. Resolver
    
    → 권한 없는 DNS 서버
    
    →DNS 클라이언트 요청을 네임 서버로 전달하고 찾은 정보를 클라이언트에게 제공하는 기능을 수행한다

### Time To Live

네트워크에서 데이터의 유효 기간을 나타내는 방법

TTL 사용 이유

→ 라우터에서 라우터로 무한정 전달될 위험성을 막기 위해

→ 패킷이 무한으로 순환하는 것을 막아줌

→ 패킷이 목적지를 찾지 못하고 네트워크 상에 잔류하는 것을 막기 위해

TTL방식

→ 라우터같은 네트워크 장비를 거칠 때 마다 값이 1씩 감소 → TTL이 0이 되면 패킷 드랍

DNS와 TTL

TTL값은 라우터 수가 아닌 DNS캐시를 유지하는 시간을 나타냄

TTL이 300이라면 300초 동안 캐싱한 정보를 유지하고 이후 같은 질의에 대한 빠른 응답 가능

|TTL이 짧을 때|TTL이 길 때|
|---|---|
|변경 사항이 빠르게 반영됨|변경 반영이 느림|
|DNS 조회가 자주 일어나 트래픽 증가|DNS 조회 수 적어서 빠르고 안정적|
|테스트나 이전 작업에 유리|안정적인 사이트 운영에 유리|
