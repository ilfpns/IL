# 내부망 (LAN)

→ 일정 조직 내에서 인터넷이 아닌 내부 네트워크를 통해 PC끼리 자원을 공유하게 하거나 그룹웨어 등을 사용할 수 있게 하는 근거리 통신망(LAN)

- 조직의 보안 정책에 따라 접근 제어 및 보안 기능을 강화해 외부 접근과 유출을 방지
- 사설 IP주소 범위를 사용하여 내부망의 IP주소를 할당함
- 내부망은 인터넷에 직접 연결되지 않을 수 있다

# 외부망 (WAN)

→ 일정 조직을 넘어 정보를 교환할 수 있는, 즉 인터넷을 통한 네트워크

- 인터넷 서비스 제공자(ISP)를 통해 접속되며 인터넷을 통해 정보를 교환하고 서비스 가능
- IP주소를 사용하여 각각의 장치나 네트워크를 전 세계적으로 식별하고 통신함

### 차이점

|내부망|외부망|
|---|---|
|사설 IP|공인 IP|
|조직간의 자원 공유를 위해 사용됨|조직, 기업, 개인의 정보 교환과 서비스를 위해 사용됨|
|조직에서 직접 보안함|개인의 컴퓨터에서 방화벽으로 보호|

### 추가 개념

**DMZ**

→ 외부에 서비스 제공 시, 내부 자원을 보호하기 위해 내부망과 외부망 사이에서 접근제한을 수행하는 영역

- 침입차단시스템 등으로 접근 제한 등을 수행하지만 외부망과 직접적인 접근이 가능하다
- 외부망, 내부망과는 조금 다는 범주의 개념으로 내부망과 함께 쓰이며 방화벽 역할을 수행하기 위한 공간이다

### Internet Protocol Version 4

→ ARPANET 기반으로 도입된 32bit 주소 체계의 핵심 인터넷 프로토콜이다.

- `x.x.x.x` 형태인 점-10진법 표기를 사용한다
- 32bit, 즉 4개의 옥텟로 구성되어 약 42억 개의 고유 주소를 제공한다

**네트워크 vs 호스트 vs 브로드캐스트 주소**

IPv4 주소는 네트워크 파트와 호스트 파트로 나뉜다

- 네트워크 → 해당 네트워크 전체 식별
- 호스트 → 네트워크 내 개별 장치
- 브로드캐스트 → 네트워크 내 전체에게 메시지 송신에 사용

**IP 주소의 클래스**

|클래스|시작 주소|범위|용도|
|---|---|---|---|
|A|0.0.0.0|~ 127.255.255.255|대형 네트워크|
|B|128.0.0.0|~ 191.255.255.255|중형 네트워크|
|C|192.0.0.0|~ 223.255.255.255|소형 네트워크|
|D|224.0.0.0|~ 239.255.255.255|멀티캐스트|
|E|240.0.0.0|~ 255.255.255.255|실험용|

하지만 32bit 주소의 한계 → 2011년 전 세계 주소가 고갈되어 IPv6가 개발됨

### Internet Protocol Version 6

새로운 헤더 포맷

- 헤더를 고정 길이로 변경
- 패킷 단편화 관련 필드가 삭제
- 체크섬 필드 삭제

향상된 서비스의 지원

- 트래픽을 효과적으로 분류할 수 있는 기능

보안

- IPv4는 IPSec이라는 보안 프로토콜을 별도로 설치해야 했었음
- IPv6는 프로토콜 내에 보안 고나련 기능을 탑재
- 확장헤더를 통해 네트워크 계층에서의 종단간 암호화 제공

| 항목             | IPv4                           | IPv6                            |
| -------------- | ------------------------------ | ------------------------------- |
| **주소 길이**      | 32비트 → 약 42억 개 주소              | 128비트 → 3.4×10³⁸ 주소             |
| **주소 표현 방식**   | 점-10진 구분(예: 192.168.1.1)       | 16진수 8블록 콜론 구분(예: 2001:db8::1)  |
| **주소 공간**      | 공인 + 사설 IP (RFC 1918)          | 글로벌 유니캐스트, 링크-로컬, 유니크 로컬 등      |
| **클래스 구분**     | A ~ E 클래스 존재                   | 클래스 없음; CIDR 기반 서브넷팅            |
| **헤더 구조**      | 20–60바이트, 필드 많음, 체크섬 존재        | 고정 40바이트, 확장 헤더 방식, 체크섬 없음      |
| **주소 구성 방식**   | 수동 설정 또는 DHCP 필요               | SLAAC(자동 구성), DHCPv6 가능         |
| **통신 방식**      | 유니캐스트, 브로드캐스트, 멀티캐스트           | 유니캐스트, 멀티캐스트, 애니캐스트 (브로드캐스트 없음) |
| **보안 기능**      | IPsec 선택사항, ARP 취약점 존재         | IPsec 기본 지원, NDP 보안 강화          |
| **NAT 필요성**    | NAT 적용이 일반적                    | NAT 불필요 (글로벌 주소 충분)             |
| **라우팅 효율성**    | CIDR 기반으로 제한적 (라우팅 테이블 증가)     | 대규모 서브넷, 계층적 집계로 처리 효율          |
| **IPv4 고갈 상태** | 2011년 이후 주소 고갈, NAT 등 우회 방식 적용 | 고갈 문제 없음, 미래 지향적 설계             |
