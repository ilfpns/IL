| Static (정적 IP)
- 한국에는 할당=되어있는 IP 갯수가 정해져있다
- 항상 동일한 IP 주소를 가진다, 네트워크에 다시 연결될 때마다 동일한 IP 주소를 유지함
- 네트워크 관리자가 직접 설정해야 함, 사용자가 IP주소를 기억하거나 기록해야 함
- 네트워크 장치가 일정한 IP주소를 유지하므로 안정적이다

# 장점
- 지정된 서버 설정에 유리하다
- 접근 권한 관리가 쉽다
- 네트워크 장비 간의 안정적 통신이 가능하다

# 단점 
- 네트워크 환경이 바뀔 때, 일일이 IP주소를 재설정 해야 한다
- 예측이 가능해, 외부 공격에 노출될 수 있다
------------------------------------------------------------------------------------------------
|DHCP (동적 IP)
동적 IP는 유동적, 즉 움직이는 IP이다
공유기에서 IP를 받는다
정적 IP는 정적으로 움직이지 않는다
아무리 시간이 지나도 IP는 바뀌지 않음

- 자동 IP 할당, 사용자가 직접 설정하지 않아도, 네트워크에 연결되면 자동으로 IP주소를 할당받는다
- 동적 IP주소, IP 주소는 동적으로 할당되며, 시간이나 어떤 이벤트에 따라 IP가 바뀔 수 있다
- 여러 장치가 네트워크에 접속하거나 나가는 경우에도 쉽게 관리할 수 있다

# 장점
- 자동 IP 설정
- 대규모 네트워크에 적합
- 한정된 주소를 효율적으로 쓸 수 있다

# 단점
- 일정 IP주소가 필요한 장치에서는 적합하지 않을 수 있다
- 고정 IP를 부여하려면, 추가적인 설정이 필요하다

|UUID
=> 네트워크 상에서 중복되지 않는 ID를 만들기 위한 표준 규약
=> 계속해서 네트워크 ID를 만들어도 중복될 확률이 0에 가깝다

- UUID : 8 - 4 - 4 - 12형태의 문자이다
- 각 문자는 16진수 숫자이다
ex) eaaa2618-b2aa-4bb8-9721-0ef0a2400f77
