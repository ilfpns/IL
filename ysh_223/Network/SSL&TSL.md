### 대칭키

⇒ 하나의 키를 사용하여 데이터를 암호화하고 복호화하는 방식

- 속도가 빠름
- 한번 유출되면 모든 보안이 해제됨

### 비대칭키

⇒ 두 개의 키를 사용하여 데이터를 암호화하고 복호화하는 방식

- 공개키가 데이터를 암호화한다
- 비밀키가 데이터를 복호화한다
- 상대적으로 느리다

> **Secure Sockets Layer**

---

⇒ 웹 브라우저와 웹 서버 간의 데이터 통신을 암호화 하는 프로토콜

> **TLS**

---

⇒ SSL의 업그레이드 버전이다

- SSL/TLS는 HTTPS 프로토콜에 사용된다

> **SSL/TLS**

---

### 통신 과정

1. SSL은 공개키와 대칭키를 혼합해서 사용한다
    
    - 키
        - 서버의 **Public Key (공개키)**
        - 서버의 **Private Key (비밀키)**
        - **Session Key (대칭 키)**
            - 진화 전: Master secret (셀 1형태), Pre Master secret (셀 2형태)
            - 진화 후: Session Key (셀 3형태)
2. 악수 > 전송 > 세션 종료
    
    1. HandShake : 상대방이 존재하는지, 데이터를 주고 받기 위해 어떤 방법을 사용할지 파악
    2. 전송 : 데이터 전송 (암호화하고 복호화하는 방법을 정함)
    3. 세션 종료
