## Subnetting [서브네팅] [[Network]]

⇒ 네트워크 자원을 효율적으로 분배하는 것

⇒ 네트워크 성능을 보장하는 것 (네트워크 영역과 호스트 영역을 쪼갬)

- **효율적 분배란**
    
    네트워크 영역과 호스트 영역을 분할 하는 것
    

**왜 사용할까?**

⇒ 너무 큰 브로드캐스트 도메인은 네트워크 환경에서 패킷 전송을 느리게 할 수 있다

- **캐스트?**
    
    하나에 송신 PC에서, 같은 IP에 연결된 PC에 대해
    
    - 멀티캐스트 : 1개의 송신, 선택적 다수 수신
    - 유니캐스트 : 1개의 송신, 1개의 수신
    - 브로드캐스트 : 1개의 송신, 모두 수신

⇒ IP는 32자리 2진수로 표한함 (2^32) 그렇기에 IP 개수가 유한하므로, 주소를 아껴야 함

⇒ 필요한 네트워크 주소만 호스트 IP로 할당 할 수 있게 만들어 네트워크 낭비를 방지함

- **IP의 특징**
    - 하나의 IP는 네트워크 부분 + 호스트 부분으로 구성된다
    - LAN 통신 → 네트워크가 같아야 함
    - LAN 내에서의 통신 → 호스트(IP의 고유 번호)로 구분
    - **EX**
        - 192.168.0.3
        - 192.168.0.4
        - Network : 192.168
        - Host : 3, 4

## SubnetMask

⇒ 할당된 IP 개수에서 필요한 IP를 제외한 낭비되는 IP를 비활성화 하여 IP 낭비를 방지하는 것

**특징**

⇒ 1과 0으로 이뤄지며, 1이 연속적으로 나와야 한다

**EX**

- 192.168.0.1/24 라는 IP가 존재함
    
    - C class 에 속함
    - default mask : 255.255.255.0
    - 255.255.255.0 = 1111 1111. 1111 1111. 1111 1111. 0000 0000
    - 1의 개수 : 24
    
    ⇒ 사용자에게 0이 표현된 부분만 호스트 IP를 할당할 수 있게 함
    
    ⇒ 192.168.0.1 ~ 192.168.0.244 까지 사용가능
    
    - **class?**
        
        ⇒ 각 기관마다 부여받는 Class가 있음
        
        |255.|255.|255.|000|
        |---|---|---|---|
        |A|B|C||
        
        - A : 255.0.0.0
            - 국가, 글로벌 기업
        - B : 255.255.0.0
            - 대기업, 대학교, 정부
        - C : 255.255.255.0
            - 중소기업, 소규모 연구소
        
        ⇒ A > B > C 순서로 많은 IP를 사용할 수 있음
        

**Subnetting은 SubnetMask와 무슨 상관관계일까?**

- Subnetting : 네트워크와 호스트를 나눈다고 정의함
    
    ⇒ 192.168.0 (네트워크) & .1 (호스트)
    

⇒ Class로 IP를 비활성화 하는 데에는 한계가 존재함

255.255.255.0 (C)는 254개의 IP를 사용해야 함, 하지만 실제로 20개의 IP만 사용한다면?

→ 234개의 IP 손해

**해결**

⇒ 네트워크와 호스트를 나눠서 호스트 값을 변경

255.255.255.0 → 255.255.255.224
