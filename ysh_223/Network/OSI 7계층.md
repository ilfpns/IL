## OSI 7계층이란?

- OSI 7계층은 네트워크 통신이 일어나는 과정을 7단계로 나눈 것을 말한다.

OSI 7계층을 나눈 이유는?

→ 통신이 일어나는 과정을 단계별로 파악할 수 있기 때문에

1. 사람들이 이해하기 쉽다.
2. 흐름을 한눈에 알기 쉽다.
3. 이상이 생긴 단계만 수정하면 된다.

|1 계층|물리 계층|Physical Layer|
|---|---|---|
|2 계층|데이터 링크 계층|DataLink Layer|
|3 계층|네트워크 계층|Network layer|
|4 계층|전송 계층|Transport Layer|
|5 계층|세션 계층|Session Layer|
|6 계층|표현 계층|Presentation Layer|
|7 계층|응응 계층|Application Laye|

## 물리 계층 (Physical Layer)

이 계층에서는 주로 전기적, 기계적, 기능적인 특성을 이용해서 통신 케이블로 데이터를 전송한다.

> 특징

- 이 계층에서 사용되는 통신 단위는 비트이며 이것은 1과 0으로 나타내어진다 → 전기적 on/off이다.
    
- 데이터를 전달하거나 받기만 할뿐 데이터가 무엇인지, 어떤 에러가 있는지 신경 쓰지 않는다.

## 데이터 링크 계층 (DataLink Layer)

물리계층을 통해 송수신되는 정보의 오류와 흐름을 관리하여 안전한 정보 전달을 수행할 수 있도록 함

> 특징

- 통신에서의 오류를 찾아주고 재전송도 함
    
- 맥 주소를 가지고 통신한다
    
    - 맥 주소는?
        
        네트워크 상에서 서로를 구분하고 인식하기 위한 일종의 주소
        
- 전송되는 단위를 프레임이라 칭함
    
- 브릿지나 스위치를 통해 맥주소를 가지고 물리계층에서 받은 정보를 전달
    

Point to Point 방식으로 신뢰성있는 전송을을 보장함

주소 값은 물리적으로 할당 받음

→ 네트워크 카드가 만들어질 때 맥 주소가 정해져 있다는 뜻

- 네트워크 카드는?
    
    컴퓨터가 네트워크와 통신하기 위해 사용하는 하드웨어 장치
    

→ 주소 체계는 계층이 없는 단일 구조이다

## 네트워크 계층 (Network Layer)

데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능

경로를 선택하여 주소를 정하고 경로에 따라 패킷을 전달해주는 것

> 특징

- 네트워크 계층은 여러개의 노드를 거칠때마다 경로를 찾아주는 역할을 하는 계층이다
- 데이터를 연결하는 다른 네트워크를 통해 전달함으로써 인터넷이 가능함
- 트래픽의 우선순위나 지연 시간을 고려하여 전송 품질을 조절할 수 있습니다. 이를 서비스품질(Qos)라 한다

## 전송 계층 (TransPort Layer)

통신을 활성화하기 위한 계층으로 보통 TCP프로토콜을 이용하며 포트를 열어서 응용프로그램들이 전송을 할 수 있게 한다.

**전송 계층이 패킷들의 전송이 유효한지 확인하고 전송 실패한 패킷들을 다시 전송한다**

- TCP
    
    ## TCP (Transmission Control Protocol)
    
    인터넷에서 데이터를 안전하게 전달하기 위한 프로토콜
    
    > 특징
    
    - **신뢰성 있음**
        
        → 데이터를 보냈는지, 제대로 도착했는지 확인한다
        
    - **순서 보장**
        
        → 먼저 보낸 데이터가 먼저 도착하게 해준다.
        
    - **에러 검사**
        
        → 데이터가 손상되었는지 검사하고, 이상하면 다시 보내달라고 한다
        
    - **연결 지향형**
        
        → 데이터를 주고받기 전에 먼저 연결부터 한다.
        

> 특징

- 종단간(end-to-end) 통신을 다루는 최하위 계층으로 종단간 신뢰성 있고 효율적인 데이터를 전송하며,
    
    기능은 오류검출 및 복구와 흐름제어, 중복검사 등을 수행한다.
    
- 양 끝단(End to end)의 사용자들이 신뢰성있는 데이터를 주고 받을 수 있도록 하며 상위 계층들이 데이터 전달의 유효성이나 효율성을 생각하지 않도록 해준다.
    
- 데이터가 들어온다면 4계층에서 해당 데이터를 하나로 합쳐서 5계층으로 넘겨준다.
    

## 세션 계층 (Session Layer)

데이터가 통신하기 위한 논리적인 연결을 말한다.

→ 진행방식

1. 연결 시작
2. 세션 유지
3. 연결 종료

> 특징

- **동시 송수신 방식(duplex),**
- **반이중 방식(half-duplex)**
- **전이중 방식(Full Duplex)**
- 체크 포인팅과 유휴, 종료, 다시 시작 과정 등을 수행한다.
- 이 계층은 **TCP/IP 세션을 만들고 없애는 책임**을 진다.

## 표현 계층 (Presentation Layer)

코드 문자 등을 번역하여 일관되게 전송하고 압축, 해제, 보안 기능을 담당한다.

> 특징

- 코드 간의 번역을 담당하여 사용자 시스템에서 데이터의 형식상 차이를 다루는 부담을 응용 계층으로부터 덜어 준다
- 인코딩이나 암호화 등의 동작이 이루어진다.
    - EBCDIC **→** ASCII
- 해당 데이터가 TEXT인지, 그림인지, GIF인지 JPG인지의 구분 등이 표현 계층의 역할이다.

## 응용 계층 (Application Layer)

사용자에게 친화적인 마지막 페이지로써 HTTP, FTP, SMTP, POP3, IMAP, Telnet 등과 같은 프로토콜이 있다.

→ 일반적인 응용 서비스 수행

> 특징

- 사용자가 사용하는 응용 프로그램과 직접 연결됨
- 파일 전송(FTP), 이메일(SMTP), 웹서비스(HTTP), 원격 접속(SSH) 등의 기능을 제공
- 개발자나 소프트웨어가 네트워크 기능을 쉽게 사용할 수 있도록 API 형태로 기능 제공 가능
- 클라이언트의 요청(Request)에 대해 서버가 응답(Response)을 주는 방식으로 동작

### 브리지 / 스위치

- 브리지와 스위치는 두 시스템을 연결하는 네트워킹 장치이다
- 두 개의 LAN을 연결하여 훨씬 더 큰 LAN을 만들어 준다

|브리지|스위치|
|---|---|
|소프트웨어 방식|하드웨어 방식|
|저속|고속|
|포트들이 동일한 속도로 전송|포트들마다 다른 속도로 전송|
|2 ~ 3개 포트|수백 개 이상 포트|
|목적지 주소를 기준으로 1:1 연결|목적지 주소를 기준으로 1:N 연결|
|데이터를 전부 받은 후 처리하는||
|Store and Forwarding방식 사용|Store and Forwarding + Cut through 방식인|
|Fagment Free 방식을 사용||
