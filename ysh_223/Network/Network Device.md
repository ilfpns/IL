## Switch

⇒ 데이터를 목적지 장치가 연결된 포트로 전달하는 장치

**특징**

- L2에 속한다
- MAC주소 테이블을 가진다

**Flow**

1. MAC 주소 학습
2. MAC 주소 테이블 제작 ( port1 = A, port2 = B ….)
3. A에서 B로 데이터 전달 (B에 해당하는 port2 만 개방)

- 플러딩
    - 만약 MAC 주소 테이블에 매칭되지 않는 주소라면?
        1. 송신자를 제외한 모든 prot에 데이터 전송
        2. 수신한 prot는 ACK를 스위치에 전송
        3. 스위치는 해당 MAC주소를 테이블에 매칭

**왜 사용할까?**

- 독립 통로
    
    ⇒ 각 포트마다 전용 통신로를 만들어주기 때문에 각각의 기기가 독립적으로 동시에 통신할 수 있다
    
- 충돌 방지
    
    ⇒ 데이터 이동 간의 충돌을 방지한다

## Hurb

⇒ 수신된 데이터(신호)를 연결된 모든 포트로 전송하는 기기

**특징**

- L1에 속한다
    
- 반이중 통신
    
    ⇒ 한 번에 한 명만 보낼 수 있다
    

**Flow**

1. 특정 포트로 데이터(신호)가 들어옴
2. 송신 포트 제외 모든 포트로 데이터 전송
3. 수신된 장치에서 본인 데이터가 아니면 버리고, 주인이라면 취한다

**왜 사용할까?**

- 단순 연결
    
    ⇒ 적은 수의 기기를 저렴하게 연결 가능
    
- 네트워크 모니터링
    
    ⇒ 모든 포트에 데이터가 흐르는 특성을 이용해, 통신 전체를 감지하는 용도로 사용됨

### **Router**

⇒ 서로 다른 네트워크 간에 최적의 경로를 찾아 데이터를 전송하는 기기

**특징**

- L3에 속한다.
- IP 주소를 기반으로 데이터를 전달한다.
- 라우팅 테이블을 참조해 경로를 결정한다.

**Flow**

1. 경로 수집
    
2. 경로 결정
    
3. 라우팅 테이블 기록
    
    ⇒ 최적의 경로를 메모리에 저장한다
    
4. 포워딩
    
    ⇒ 데이터 패킷이 들어오면 테이블을 보고 알맞은 인터페이스로 보내준다
    

**왜 사용할까?**

- 네트워크 간 연결
    
- 경로 최적화
    
    ⇒ 데이터가 길을 잃지 않고 가장 빠르게 도착하게 한다
    
- 브로드캐스트 제어
    
    ⇒ 불필요한 전체 전송(Broadcasting)이 다른 네트워크로 넘어가지 않게 차단하여 트래픽을 관리한다.

### **Modem**

⇒ 컴퓨터의 디지털 신호와 외부 통신 회선의 아날로그 신호를 상호 변환해주는 기기

**특징**

- L1에 속한다.
- 변조(Modulation)와 복조(Demodulation) 기능을 수행한다.

**Flow**

1. **신호 수신:** 외부(광케이블, 전화선 등)에서 들어오는 아날로그 신호를 받는다
2. **복조(Demodulation):** 아날로그 신호를 컴퓨터가 읽을 수 있는 0과 1의 **디지털 신호**로 변환한다
3. **신호 송신:** 반대로 컴퓨터의 디지털 신호를 외부로 보낼 때는 아날로그 신호로 변조(Modulation)하여 내보낸다

**왜 사용할까?**

- 신호 체계 일치
    
    ⇒ 전화선이나 케이블TV 망은 데이터를 전기로 실어 나르는 방식이 컴퓨터와 다르기 때문에
    
- 인터넷 접속
    
    ⇒ 집 안의 네트워크 기기들이 전 세계 인터넷 망에 접속할 수 있게 입구 역할을 해준다
