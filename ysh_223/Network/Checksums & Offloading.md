### Checksum [[Network]]
- 저장장치상에서의 데이터 일관성을 확인하기 위한 수단

- 의미 : 데이터 무결성 검사용 값
- 역할 : 전송 중 비트 오류 검출

네트워크 대부분은 데이터 전송 시에 발생되는 에러로 인해 일관성을 해친다

1. 데이터 기록 전 일관성 확인이 필요한 바이트 스트림의 마지막 부분에 체크섬값을 계산하여 추가함
2. 데이터 기록 후 읽어 들이는 시점에 장치상의 바이트 스트림값을 토대로 계산한 체크섬 값과의 일치여부를 확인

### CheckSum Offloading
- 의미 : 체크섬 계산 & 검증을 CPU가 아니라 NIC가 수행
- 목적 : CPU부하 감소, 패킷 처리 성능 향상
- NIC?
	네트워크에 연결하는 하드웨어 인터페이스
	이더넷 포트가 달린 칩/카드

**과정**
- 송신
	1. CPU/커널 패킷 생성
	2. 체크섬 계산 스킵
	3. 체크섬 계산을 요구하는 메타데이터 추가
	4. DMA로 NIC에게 전달
	5. NIC가 체크섬 계산
	6. 계산된 체크섬을 헤더에 삽입
	7. 물리망으로 전송

**수신**
	1. NIC가 패킷 수신
	2. NIC가 체크섬 검증
	3. 결과(OK/ERR)만 메타데이터로 표시
	4. DMA로 메모리에 전달
	5. 커널은 결과만 보고 처리

**사용 이유**
1. CPU 병목 제거
2. NIC는 병렬 전용 하드웨어
	- 체크섬용 연산 유닛 내장
	- 파이프라인 처리
3. 전체 시스템 성능 증가
