# MAC

→ 단말이 가지고 있는 물리적인 주소

### 특징

- 논리적 IP와 다르게 주소 변경이 어려움
- 네트워크 인터페이스가 가지는 고유 식별자이다
- IP로만 통신시 원활하지 않아 혼선이 온다
    - 언제든 논리적 주소로 바뀔 수 있기 때문에 → 고정적 주소 MAC 필요

MAC주소는 혼자만 가지고 있는 고유의 번호이다.

→ 이 IP와 MAC을 연결 시켜즈는 프로토콜이 ARP이다

### 구성

- 48비트로 구성됨
- 16진수로 표기함 (00:1A:2B:3C:4D:5E)
- 앞의 3바이트 : 제조사 고유 번호
- 뒤의 3뱌이트 : 장치 일련 번호

### 사용 범위

- 같은 LAN내에서만 사용됨
- 라우터를 넘으면 MAC 주소가 의미 없음 (→ WAN에서는 사용 불가)

# ARP

→ 주소 결정 프로토콜로 IP주로를 물리적 주소인 MAC주소로 연결시켜주는 프로토콜

### ARP의 필요성

- IP주소를 사람이 인식하고 관리하기 좋은 논리적 주소임
- 실제 주소는 MAC주소를 통해 이루어지기에 둘을 연결해주는 과정 필요

### ARP캐시

- 한 번 ARP로 얻은 MAC주소는 일정 시간 캐시에 저장
- 매번 요청하지 않기에 성능이 향상됨

### ARP통신 과정 (2단계)

- ARP Request : 상대의 IP주소를 가지고 MAC 주소 정보를 요청하는 패킷
    
    - 이는 LAN구간 모든 호스트에게 전달해야 하기 때문에 Brodcast 통신을 한다
- ARP Reply : ARP Repuest를 받은 호스트가 해당 IP를 가지고 있으면 자신의 MAC주소를 담아 응답하는 패킷
    
    - ARP Request를 요청한 호스트에게만 전달하면 되기에 호스트에게 Unicast통신을 한다
    - 다른 IP의 ARP Rrequest는 무시한다
